The ask: Build a new elementary/middle school simulation software for a physical, hands-on experience where students operate businesses, manage money, and fulfil civic roles in a simulated town for a day. The new platform replaces the existing system with a modern, tablet-first, real-time experience.
What we're building

Core Platform

A unified digital platform supporting a real-economy simulation — banking, payroll, point-of-sale, voting, marketing, and civic engagement — running simultaneously across 80–150 student devices in a physical facility.

Key Design Principle

Real Feels Real

Tap-to-pay, live dashboards, AI-assisted marketing, digital banking — the UX should mirror tools students will encounter in the real world, but age-appropriate and guided.

Economic Model — How the Town Works
Businesses (14–20)

Each storefront has 6–8 student employees. Apply for loans, manage payroll, run POS, serve customers, send/receive B2B invoices.

Banking & Finance

Students hold personal checking + savings accounts. Debit cards, QR-code payments, NFC tap-to-pay. Real-time balance sync across all devices.

Civic Layer

Voting on town issues, tax mechanics, philanthropic scenarios, community leaderboard — students see how decisions affect the whole economy.

Learning Objectives
Delivery Modes
Permanent Facility

Full onsite JA BizTown installation in a dedicated physical space.

Mobile / Pop-Up

Tablet-based "JA BizTown Mobile" deployed in schools, community centres.

Paper Fallback — Required

Complete paper-based version for power/internet outages.

Section 04
Required Software Modules
Every module below is a hard requirement. The platform must support all simultaneously on a one-device-per-student model with real-time data sync across 80–150 devices.
💳
Point of Sale
Price per storefront, profit/loss slider vs loan. Real-time transaction recording.
📲
Student Payment
QR code per device. Colour-coded shopping break state. Debit card + NFC tap-to-pay.
🎨
Digital Marketing
Canva-style tool built in. Share designs across devices, roles, businesses. AI slogan gen.
🏦
Banking System
Personal checking + savings. Deposits, withdrawals, real-time balance sync.
💰
Payroll
Direct deposit or printed checks. 2nd paycheck as triggered spot bonus.
📋
Loan System
CFO applies on shared screen. CEO finalises at bank. Screen-share view for team.
📬
Billing & Bill Pay
Send/receive B2B invoices. Transaction tracking. Overdue alerts.
🗳️
Voting
Kiosk + mobile Mayor version. Results feed leaderboard and State of the Town.
🌪️
Business Scenarios
Randomised ethical dilemmas. CEO decides. XP gain/loss reflects community impact.
🏆
Leaderboard (XP)
Live, large-screen ready. Loan speed, profit, donations, surveys, voting all score.
📊
Business Dashboard
P&L, profit calculator, competitor pricing, loan status, pie charts + bar graphs.
📻
Radio Business
Integrated with facility A/V. Auto-imports slogans for DJ to broadcast on-air.
📝
Surveys & Evaluation
Pre + post. 4–6 locked national questions. Areas add up to 6. Min 10 items, 125-char prompts.
🤝
Team Building Widget
One-device-per-user real-time collaboration. Each input visibly contributes to shared output.
🤖
AI Features
Slogan generation for marketing. CEO speech transcription via voice-to-text.

Section 05
Audience Roles & Requirements
Role Composition per Simulation Day
Students — Primary Users
  • Grades 5–6, ages 9–12
  • One device per student (tablet)
  • Roles: CEO, CFO, CMO, general employees
  • Apply for loans, submit payroll, vote, shop
  • Auth: QR code or PIN — not device-bound
  • English + Spanish UI at minimum
Simulation Manager
  • Full-time JA staff, leads entire day on mic
  • Centralised dashboard: real-time town status
  • Push/pause simulation phases from laptop
  • Reverse transactions, reissue paychecks, move students remotely
  • Award XP and subsidies to businesses
  • Trigger activities (voting, scenarios, payroll)
Volunteers
  • Corporate, parent, or community volunteers
  • Tablet dashboard: P&L, student progress, loan status
  • Integrated guide with discussion prompts
  • Clear indicators for when + why to intervene
  • End-of-day feedback survey
Teachers
  • Deliver 12 pre-sim lessons before simulation day
  • Monitor behaviour and engagement day-of
  • End-of-day reports via QR code or simple login
  • Individual student, business, and full simulation reports
  • May serve as improvised volunteer if needed
CMS Admin Roles
Super Admin — National

JA USA. Sets master templates, locks evaluation questions, pushes updates to all JA Areas without overwriting local customisation.

Local Admin

Per JA Area. Customises businesses, sponsors, logos, payroll rates, tax rates, loan terms. Restricted to assigned facilities only.

Simulation Manager

Day-of setup and activation. Creates templates, adjusts timing, enables/disables modules, makes real-time CMS changes during live sim.
